# 用户快速开始

## 1. 打开系统

### Windows 普通用户最简单打开方式

收到 Windows 试用包后：

1. 解压 zip。
2. 打开解压后的文件夹。
3. 双击 `PropertyFeeSystem.exe`。
4. 浏览器自动打开后，用默认账号登录。

如果 Windows 安全提示拦截，请点击「更多信息」→「仍要运行」。

### macOS 普通用户最简单打开方式

收到 macOS 试用包后：

1. 解压 zip。
2. 双击 `物业管理收费系统.app`。
3. 如果系统拦截，请进入「系统设置 → 隐私与安全性」点击「仍要打开」。


如果收到的是免安装包，请打开整个 `PropertyFeeSystem` 文件夹，然后双击：

```text
PropertyFeeSystem.exe
```

如果收到的是源码脚本版，请双击：

```text
start_windows.bat
```

系统会自动打开浏览器登录页。

## 2. 默认账号

```text
用户名：admin
密码：admin123
```

正式使用前，请让管理员进入「操作员管理」修改默认密码，并按岗位设置账号权限。

本地版不需要注册互联网账号；这里的账号只是本机系统内部账号。单人试用可以先只用 admin，正式收费建议至少创建一个财务收费账号，如需客服查看催费对象，再创建客服只读账号。

## 3. 第一次使用顺序

1. 进入「数据导入」，下载基础资料模板。
2. 按模板填写楼栋、房号、层高、面积、业主信息、合同日期等基础信息。
3. 回到「数据导入」，预览识别结果，确认后导入。
4. 进入「收费项目」，检查物业费、电梯费、水费、公摊费等收费项目。
5. 进入「账单管理」→「生成账单」，先预览，再确认生成。
6. 进入「物业收费」或账单详情进行收费登记。
7. 收费后可以打印收据，并在「对账报表」核对当日和当月数据。

## 4. 日常收费人员主要入口

| 场景 | 入口 |
|---|---|
| 导入房间/业主资料 | 数据导入 |
| 设置收费规则 | 收费项目 |
| 批量生成应收账单 | 账单管理 → 生成账单 |
| 收取费用 | 物业收费 |
| 打印收据 | 账单详情 / 收据打印 |
| 查当天收了多少 | 对账报表 |
| 月末锁账 | 期末结账 |

## 5. 数据保存在哪里

macOS 默认保存到：

```text
~/Library/Application Support/PropertyFeeSystem
```

Windows 默认保存到：

```text
%APPDATA%\PropertyFeeSystem
```

重要文件：

- `property.db`：业务数据库。
- `backups`：自动备份。
- `startup_error.log`：启动失败日志。

如果在同一台 Mac 上看到旧测试数据，说明系统读取了本机用户数据目录里的历史数据，不是交付包混入了数据库。需要空白试用时，请先关闭 app，再双击交付包里的 `清空本机试用数据.command`。脚本会先备份旧数据到桌面，再清空本机试用数据。

## 6. 出问题先看这里

- 打不开系统：运行 `diagnose_windows.bat`，把窗口内容发给维护人员。
- Excel 导入报错：先运行 `install_windows_dependencies.bat`。
- 金额不确定：不要直接入账，先在预览页或账单详情核对，再由收费人员确认金额。
- 客服账号只能查看房间、账单和催费对象，不能修改财务数据。

## 7. 桌面窗口按钮说明

| 按钮 | 用途 |
|---|---|
| 打开系统 | 重新打开浏览器收费系统页面。 |
| 打开数据目录 | 查看本机数据库、备份和日志文件。 |
| 查看诊断信息 | 显示 Python、数据目录、数据库、依赖状态，并保存 `runtime_status.log`。 |
| 打开错误日志 | 打开 `startup_error.log`，用于排查启动失败。 |
| 退出 | 关闭桌面窗口并停止本地服务。 |
