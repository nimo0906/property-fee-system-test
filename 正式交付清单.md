# 正式交付清单

本清单用于最后一轮上线前收口，覆盖 UI 细节收口、文档 / 使用说明、Windows 打包准备、正式交付文件和安全边界。完成本清单后，才进入正式发给用户试用或部署。

## 1. UI 细节收口

- [ ] 首页、房间管理、收费项目、物业收费、自动出账、账单管理、缴费记录、发票管理、对账报表、操作员管理页面可正常打开。
- [ ] 页面标题、主要按钮、返回按钮、空状态提示语清楚，不出现测试文案或看不懂的入口。
- [ ] 发票管理默认显示全部可开发票账单；选择月份后保留所选账期。
- [ ] 缴费记录进入打印或收据页面后，返回应回到缴费记录页。
- [ ] 隐藏模块不出现在菜单，直连返回 404。

## 2. 文档 / 使用说明

- [ ] `用户快速开始.md` 面向普通用户，可按步骤打开系统、登录、导入、出账、收费、打印收据和查看报表。
- [ ] `使用说明.md` 面向管理员/维护人员，说明数据目录、备份、清空试用数据、权限和桌面按钮。
- [ ] `Windows客户试用说明.md` 面向 Windows 试用用户，不再引导已下线的业主端或支付订单。
- [ ] `交付验收清单.md` 覆盖 macOS、Windows、权限、备份、主流程、发票、报表和安全边界。
- [ ] `docs/local-security-boundary.md` 和 `docs/release-update-policy.md` 已阅读并确认。

## 3. Windows 打包准备

- [ ] 在 Windows 电脑先运行 `check_windows_packaging_ready.bat`。
- [ ] 检查通过后运行 `package_windows_release.bat`。
- [ ] 只交付 `release\windows\物业管理收费系统-v2.0-windows.zip`，不要发源码目录。
- [ ] zip 内不包含 `.env`、`property.db`、`*.db`、`backups/`、缓存和日志。
- [ ] 用户数据目录固定为 `%APPDATA%\PropertyFeeSystemData`。

## 4. macOS 交付准备

- [ ] 运行 `./build_macos_app.command` 生成 `dist/物业管理收费系统.app`。
- [ ] 双击 App 可启动桌面窗口并打开登录页。
- [ ] 用户数据目录固定为 `~/Library/Application Support/PropertyFeeSystemData`。
- [ ] 如系统拦截，按「系统设置 → 隐私与安全性 → 仍要打开」处理。

## 5. P0/P1 安全与数据检查

- [ ] 默认管理员密码正式使用前必须修改。
- [ ] 管理员、业务管理员、财务收费、客服只读权限与菜单一致。
- [ ] 备份恢复、清空试用数据、系统更新、用户管理等高风险入口受管理员权限保护。
- [ ] 备份路径校验已覆盖，不能访问备份目录外文件。
- [ ] 清空试用数据只用于演示/试用环境，真实业务数据电脑不得使用。
- [ ] P0/P1 问题已清零；P2/P3 只列入上线后优化。

## 6. 发布前必须运行

```bash
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 -m py_compile server/*.py server.py desktop_app.py desktop_runtime.py
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 -m pytest -q
python3 scripts/desktop_release_check.py
```

## 7. 最终交付文件

- [ ] macOS：`dist/物业管理收费系统.app` 或 macOS zip。
- [ ] Windows：`release\windows\物业管理收费系统-v2.0-windows.zip`。
- [ ] 文档：`用户快速开始.md`、`Windows客户试用说明.md`、`使用说明.md`、`交付验收清单.md`、`正式交付清单.md`、`数据备份说明.md`。
- [ ] 客户沟通：`Windows用户发送文案.md` 或 `客户试用说明.md`。
