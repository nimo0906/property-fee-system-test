# 物业管理收费系统 macOS 试用说明

## 一、发送给试用客户的话术

您好，附件是「物业管理收费系统」macOS 试用包。请按下面步骤打开：

1. 解压 `物业管理收费系统-v2.0-macos.zip`。
2. 打开解压后的文件夹。
3. 双击 `物业管理收费系统.app`。
4. 如果 macOS 提示无法打开，请进入「系统设置 → 隐私与安全性」，在安全提示处点击「仍要打开」。
5. 系统启动后会自动打开浏览器页面。

默认后台账号：

```text
用户名：admin
密码：admin123
```

如果这台 Mac 之前试用过旧版本，打开后可能会看到旧测试数据。需要重新空白试用时，请先关闭 app，再双击文件夹里的：

```text
清空本机试用数据.command
```

该脚本会先把旧数据备份到桌面，再清空本机试用数据。重新打开 app 后就是新的空白试用环境。

## 二、试用前提醒

- 本试用包为免安装版本，不需要联网服务器。
- 数据默认保存在当前 Mac 用户目录，不会上传到云端。
- 本版本未做 Apple 签名和公证，所以 macOS 首次打开可能会拦截，这是正常现象。
- 请不要直接删除 app 运行中的数据目录；如需清空，请使用 `清空本机试用数据.command`。

## 三、建议试用流程

请客户按下面顺序体验：

1. 登录后台首页。
2. 进入「数据导入」，查看基础资料导入入口。
3. 进入「收费项目」，查看物业费、水费、电梯费等规则。
4. 进入「账单管理」，查看账单列表和账单详情。
5. 进入「物业收费」，体验收款登记和收据入口。
6. 进入「支付订单」，查看模拟支付订单和对账状态。
7. 进入「电子票据」，查看票据申请和管理流程。
8. 进入「对账报表」，查看收入统计和对账信息。
9. 进入业主端登录页，体验业主查看房间、账单、消息、票据入口。

## 五、数据保存位置

macOS 默认数据目录：

```text
~/Library/Application Support/PropertyFeeSystem
```

主要文件：

- `property.db`：本机业务数据库。
- `backups/`：自动备份和手动备份目录。
- `startup_error.log`：启动失败日志。
- `runtime_status.log`：诊断信息。

## 六、常见问题

### 1. 双击 app 没反应怎么办？

先确认是否被 macOS 安全机制拦截。进入「系统设置 → 隐私与安全性」，找到刚才的安全提示，点击「仍要打开」。

如果仍无法打开，把以下信息发给维护人员：

- macOS 版本。
- 是否出现安全提示。
- 解压后的完整文件夹截图。
- `~/Library/Application Support/PropertyFeeSystem/startup_error.log` 内容。

### 2. 为什么打开后看到旧数据？

因为这台 Mac 之前运行过系统，旧数据保存在本机用户目录。交付包本身不包含本机数据库。

需要空白试用时，运行：

```text
清空本机试用数据.command
```

### 3. 清空数据会不会直接丢失？

不会。脚本会先备份到桌面：

```text
~/Desktop/PropertyFeeSystemDataBackups/
```

再清空本机数据目录。

### 4. 试用包可以直接正式使用吗？

可以用于内部试用和流程验证。正式对外使用前，建议再完成：

- 真实客户数据导入验收。
- 真实收费流程验收。
- 管理员密码修改。
- 数据备份策略确认。
- Windows 端实机打包验收。
- 如需对外分发，再考虑 Apple 签名、公证或安装器。
