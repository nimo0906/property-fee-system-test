# 数据备份说明

## 真实数据保护

从录入真实房间、业主、账单、收款开始，数据目录中的 `property.db` 就是正式业务数据。任何导入、生成账单、恢复备份、清空试用数据前，都必须先确认备份。

严禁把 `property.db`、`*.db`、`backups/`、`.env` 提交到 GitHub 或发给无关人员。

如需真实数据试运行，请先阅读：

```text
真实数据试运行保护方案.md
真实数据导入前验收清单.md
```


## 自动备份

系统现在会在两种情况下自动备份数据库：

1. **每次程序启动时**：生成 `auto_startup_*.db`。
2. **每日首次使用时**：生成 `auto_daily_*.db`。

自动备份保存在 `backups/` 目录。系统会自动清理旧的自动备份，每类默认保留最近 10 个。手动备份 `backup_*.db` 不会被自动清理。

## 手动备份

进入「数据备份」页面，点击「创建备份」。建议在以下操作前手动备份：

- 批量调价
- 批量导入基础资料
- 生成大批量账单
- 恢复历史备份
- 月末结账前

## 恢复备份

进入「数据备份」页面，选择备份文件，点击「恢复」。恢复会覆盖当前数据库。系统会在恢复前自动生成 `auto_before_restore_*.db`，如果恢复错了，可以再恢复该备份。

## 操作日志

管理员可进入「操作日志」查看关键操作记录，包括：

- 登录成功/失败
- 金额修改
- 账单删除
- 公摊账单生成
- 批量调价
- 缴费登记
- 数据导入
- 备份恢复/删除

金额相关操作会记录原值、新值、操作人、时间和原因。

## macOS 试用前清空本机旧数据

如果同一台 Mac 之前运行过本系统，再次打开新的试用包时，系统会继续读取本机用户数据目录中的旧数据库：

```text
~/Library/Application Support/PropertyFeeSystemData
```

这不是交付包混入了旧数据，而是桌面软件按用户目录保存本机数据。

仅在确认当前电脑没有真实业务数据、只是试用环境时，才可以在关闭系统后双击交付包内的：

```text
清空本机试用数据.command
```

该脚本会先把原数据目录备份到桌面：

```text
~/Desktop/PropertyFeeSystemDataBackups/
```

然后清空本机运行数据。再次启动 app 时，系统会自动创建新的空白试用数据库。

注意：该脚本只处理当前 Mac 用户目录下的数据目录，不会修改交付包中的 `.app` 和说明文档；一旦该目录中已经录入真实业务数据，不要使用该脚本。
