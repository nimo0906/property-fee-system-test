# 交付验收清单

## 1. Windows 启动验收

- [ ] 在 Windows 电脑上打开完整 `PropertyFeeSystem` 文件夹。
- [ ] 双击 `PropertyFeeSystem.exe` 可以启动。
- [ ] 浏览器能自动打开登录页。
- [ ] 关闭桌面窗口后，本地服务能退出。
- [ ] 如果使用脚本版，双击 `start_windows.bat` 可以启动。


## 1A. macOS 启动验收

- [ ] 在 Mac 上运行 `build_macos_app.command` 可以生成 `dist/物业管理收费系统.app`。
- [ ] 双击 `物业管理收费系统.app` 可以启动桌面窗口。
- [ ] 浏览器能自动打开登录页。
- [ ] 数据保存到 `~/Library/Application Support/PropertyFeeSystem`。
- [ ] 关闭桌面窗口后，本地服务能退出。
- [ ] 启动失败时能在数据目录找到 `startup_error.log`。

## 2. 首次启动与登录

- [ ] 默认账号 `admin / admin123` 可以登录。
- [ ] 登录后首页显示「收费工作台」。
- [ ] 管理员可以进入「操作员管理」。
- [ ] 财务收费账号不能进入「操作员管理」。
- [ ] 客服只读账号不能提交新增、修改、收费等写操作。

## 3. 基础资料与导入模板

- [ ] 「数据导入」页面可以打开。
- [ ] 可以下载导入模板。
- [ ] 按模板填写楼栋、房号、层高、面积、业主信息、合同日期后可以预览。
- [ ] 导入前有确认界面。
- [ ] 导入后有结果详情和问题行提示。

## 4. 收费主流程

- [ ] 「收费项目」能查看和新增自定义收费项目。
- [ ] 「生成账单」可以先预览，确认后才写入数据。
- [ ] 收费人员可以收费登记。
- [ ] 收费后可以打印收据。
- [ ] 「对账报表」可以查看应收、实收、欠费和收费率。
- [ ] 「期末结账」后，当期账单不能继续收费或修改。

## 5. 数据与备份

- [ ] 数据保存到 `%APPDATA%\PropertyFeeSystem`。
- [ ] `property.db` 文件存在。
- [ ] `backups` 文件夹存在。
- [ ] 导入、收费、批量修改、生成账单等关键写操作前会自动备份。
- [ ] 备份恢复前有风险确认页面。

## 6. 诊断与故障处理

- [ ] `diagnose_windows.bat` 可以输出 Python、数据目录、数据库、备份目录和缺失依赖。
- [ ] 启动失败时会写入 `startup_error.log`。
- [ ] 缺少 Excel 依赖时，运行 `install_windows_dependencies.bat` 后可恢复。

## 7. 交付文件检查

- [ ] 交付时复制整个 `dist\PropertyFeeSystem` 文件夹，不只复制 `.exe`。
- [ ] 文件夹内包含 `PropertyFeeSystem.exe`。
- [ ] 文件夹内包含 `用户快速开始.md`。
- [ ] 文件夹内包含 `交付验收清单.md`。
- [ ] 文件夹内包含 `使用说明.md`。


## 8. 内部 JSON API 验收（维护人员）

- [ ] 已阅读 `docs/api/v2-internal-json-api.md`。
- [ ] 未登录访问 `/api/v1/owners/1` 返回 JSON `401 unauthorized`，不是 HTML 登录页。
- [ ] 登录后可以访问 `/api/v1/owners/{id}`、`/api/v1/rooms/{id}`、`/api/v1/bills/{id}`。
- [ ] `/api/v1/payments/preview` 只返回收款影响，不新增缴费记录。
- [ ] `readonly` 账号不能调用 `/api/v1/payments` 写入收款。
- [ ] `/api/v1/payments` 写入前会生成 `auto_before_api_payment_*.db` 备份。
- [ ] `/api/v1/payments` 写入后 `audit_logs` 有 `api_payment_create` 记录。
- [ ] API 只在本机 `127.0.0.1` 桌面服务端口使用，未暴露到公网。
