# 物业管理收费系统桌面化优化清单

## 当前定位

当前系统已经具备本地收费管理的主要业务页面，但仍是“本地网页应用 + 开发者启动方式”。要交给普通用户使用，需要补齐桌面启动、Windows 兼容、数据目录、依赖安装、真实数据导入和操作防错。

## 已确认问题

| 优先级 | 问题 | 影响 | 当前处理状态 |
|---|---|---|---|
| P0 | 静态文件路径越界可访问数据库 | 数据泄露风险 | 已修复并加测试 |
| P0 | 只读账号可执行写操作 | 权限语义不可信 | 已限制 readonly 写操作 |
| P0 | 备份恢复后不补齐系统表 | 旧备份恢复后可能登录或页面异常 | 已恢复后执行 db_init |
| P0 | admin 初始化依赖电梯阶梯初始化分支 | 旧库可能没有管理员账号 | 已修复 |
| P1 | 没有桌面启动器 | 用户必须懂命令行 | 已新增 desktop_app.py 和 start_windows.bat |
| P1 | Windows 数据目录未分离 | 打包后写安装目录可能失败 | 已新增用户数据目录 |
| P1 | 依赖未声明 | 换机器后 Excel 功能可能直接启动失败 | 已新增 requirements.txt |
| P1 | .XLS 真实数据暂不支持 | 收款明细表无法直接导入 | 已支持解析、预览和规则分析 |
| P1 | 文档仍写旧路径和旧启动方式 | 用户按文档操作会失败 | 已更新基础启动说明 |
| P2 | 缺少一键打包脚本和图标 | 还不是最终安装包体验 | 待处理 |
| P2 | 导入前缺少预览和错误明细 | 真实数据容易导错且难追踪 | 已新增导入预览，错误明细继续加强 |
| P2 | 普通操作员权限仍较粗 | 删除/配置类操作应继续细分 | 待处理 |

## 下一步优化顺序

1. 更新使用说明，改为普通用户视角：Windows 双击启动、首次登录、数据目录、备份位置。
2. 建立收款明细正式入账：根据规则分析结果确认“垃圾费、暖气费、其它费用、押金、预收、冲抵”的最终口径，再自动写入账单和缴费。
3. 增加桌面打包方案：PyInstaller spec、图标、版本号、Windows 测试步骤。
4. 细化权限：operator 可日常收费录入，admin 才能删除、恢复、配置系统参数。
5. 增加数据安全：启动时自动备份、关闭前提示、数据库健康检查。
