# 物业管理收费系统 v2.0

## 系统简介

本系统实现**多模型独立开票**，支持以下费用类型各自出账、独立开票：

| ID | 费用项目 | 计算方式 | 单价 |
|---|---------|---------|------|
| 1 | 🏠 物业费(居民) | 面积 × 单价 | 1.9元/m² |
| 2 | 🏠 物业费(商户) | 面积 × 单价 | 5.0元/m² |
| 3 | 🛗 电梯费 | 楼层阶梯计费 | 基础1元，每5层+0.05元 |
| 4 | 💧 二次供水运行费 | 面积 × 单价 | 0.5元/m² |
| 5 | 💧 水费(非居民) | 用量 × 单价 | 5.8元/m³ |
| 6 | 💧 水费(特行) | 用量 × 单价 | 20.11元/m³ |
| 7 | ⚡ 公摊能耗费 | 按户计费 | 10元/户 |
| 8 | 🗑️ 生活垃圾费 | 按户计费 | 10元/户 |

## 快速启动（普通用户）

### Windows

第一次使用先双击：

```text
install_windows_dependencies.bat
```

之后日常使用双击：

```text
start_windows.bat
```

系统会自动打开浏览器。数据库和备份会保存在当前 Windows 用户的数据目录中，不需要手动移动 `property.db`。

### macOS / 开发测试

```bash
python3 desktop_app.py
```

如果只想启动本地网页服务，不打开桌面窗口：

```bash
python3 desktop_app.py --serve-only
```

### 默认账号

- 用户名：`admin`
- 密码：`admin123`
- 正式交付前建议进入「操作员管理」修改默认密码。
- 本地版不需要注册互联网账号；管理员可按岗位创建财务收费账号、客服只读账号。


## Windows 桌面交付说明

### 文件用途

| 文件 | 用途 |
|---|---|
| `start_windows.bat` | 日常启动入口，普通用户双击这个文件。 |
| `install_windows_dependencies.bat` | 首次使用或导入 Excel 报错时运行，用于安装依赖。 |
| `diagnose_windows.bat` | 启动失败时运行，会显示 Python、数据目录、数据库、缺失依赖和日志位置。 |
| `desktop_app.py` | 桌面启动器，会启动本地服务并打开浏览器。 |


### 交付给用户时建议附带

- `用户快速开始.md`：给一线收费人员看的最短使用说明。
- `交付验收清单.md`：给维护人员或项目负责人逐项验收。
- `使用说明.md`：完整说明。

### Windows 首次使用步骤

1. 安装 Python 3。安装时勾选 `Add python.exe to PATH`。
2. 双击 `install_windows_dependencies.bat`。
3. 双击 `start_windows.bat`。
4. 浏览器打开后，用默认账号登录：`admin / admin123`。
5. 正式使用前，进入「操作员管理」修改默认密码。

### 数据保存位置

Windows 默认保存到当前用户目录：

```text
%APPDATA%\PropertyFeeSystem
```

其中：

- `property.db`：业务数据库。
- `backups`：系统自动备份目录。
- `startup_error.log`：启动失败日志。

### 桌面窗口按钮

- 「打开系统」：重新打开浏览器页面。
- 「打开数据目录」：查看 `property.db`、`backups`、日志文件。
- 「查看诊断信息」：显示并保存运行诊断到 `runtime_status.log`。
- 「打开错误日志」：打开 `startup_error.log`。
- 「退出」：关闭桌面窗口并停止本地服务。

### 启动失败处理

1. 先双击 `diagnose_windows.bat`。
2. 如果显示 `Missing dependencies`，重新运行 `install_windows_dependencies.bat`。
3. 如果仍失败，把诊断窗口内容和 `startup_error.log` 发给维护人员。


## Windows 免安装包制作说明（维护人员）

如果要给普通用户发一个更接近桌面软件的版本，建议在 Windows 电脑上执行：

```text
build_windows_exe.bat
```

构建完成后，把整个目录发给用户：

```text
dist\PropertyFeeSystem
```

用户只需要双击：

```text
PropertyFeeSystem.exe
```

注意事项：

- 不要只复制单个 `.exe`，必须复制整个 `dist\PropertyFeeSystem` 文件夹。
- 打包后的业务数据仍保存在 `%APPDATA%\PropertyFeeSystem`，不会保存在安装包目录里。
- 如果用户启动失败，先让用户运行普通脚本版的 `diagnose_windows.bat` 或查看 `%APPDATA%\PropertyFeeSystem\startup_error.log`。
- 当前打包方案是 PyInstaller one-folder，优先保证稳定和可排查，不优先追求单文件。


## macOS 桌面交付说明

### 普通启动

开发测试或内部电脑可直接运行：

```bash
python3 desktop_app.py
```

系统会打开一个本地桌面窗口，并自动打开浏览器登录页。macOS 数据默认保存在：

```text
~/Library/Application Support/PropertyFeeSystem
```

其中 `property.db` 是业务数据库，`backups` 是自动备份目录。

### macOS App 打包（维护人员）

如需生成可双击的桌面 App，在 Mac 上运行：

```bash
./build_macos_app.command
```

构建完成后交付：

```text
dist/物业管理收费系统.app
```

用户双击 `物业管理收费系统.app` 即可启动。首次打开如遇到 macOS 安全提示，可在「系统设置 → 隐私与安全性」中允许打开，或右键 App 选择「打开」。

注意事项：

- 业务数据不保存在 App 包内，而是保存在当前用户的 Application Support 数据目录。
- 更新 App 时不要删除用户数据目录，避免误删业务数据。
- 启动失败时查看 `~/Library/Application Support/PropertyFeeSystem/startup_error.log`。

## 使用流程

### 第一步：录入房间（关键！）
添加房间时请填写：
- **楼栋/单元/房号**：如 B栋、1单元、701
- **楼层**：用于计算电梯费（7~26层）
- **类别**：选择「居民」或「商户」（决定用哪个物业费单价）
- **面积**：用于计算物业费、二次供水运行费
- **物业费单价**：如不填则用默认值（居民1.9/商户5.0），个别房间有特殊单价时可填写此项

### 第二步：添加业主
进入「业主管理」→ 添加业主，然后在房间管理中关联业主。

### 第三步：录入抄表
水费需要录入水表读数（非居民5.8元/m³，特行20.11元/m³），操作步骤：
1. 进入「抄表管理」→ 录入抄表
2. 选择房间和费用类型（水费）
3. 录入读数字，点击「确认」

### 第四步：生成账单（核心）
1. 进入「账单管理」→ 生成账单
2. 选择账期（如 2026-05）
3. 勾选需要生成的费用类型
4. 系统自动计算每户应缴金额：
   - **物业费(居民)** = 面积 × 1.9（商户 × 5.0）
   - **电梯费** = 1 + (楼层-1)//5 × 0.05
   - **二次供水** = 面积 × 0.5
   - **公摊/生活垃圾** = 每户10元
5. 如需调整金额，点击修改

### 第五步：缴费
- 单笔缴费：点击账单的「缴费」按钮
- 批量缴费：勾选多笔 → 批量缴费
- **修改金额**：如自动计算与实际不符，点击「修改金额」手动调整

### 第六步：查看报表
查看各费用类型的应收/实收/收费率，以及近12个月趋势。

### 数据导入
- 支持 `.csv`、`.xlsx`、`.xls`
- 建议先点击「预览识别结果」，确认系统识别的数据类型、列头行和字段匹配
- 普通业主、房间资料可确认导入；收款明细表只导入基础资料，不导入历史金额
- 收款明细表会先进入预览，不会直接自动入账；历史金额只用于识别收费项目，金额必须由用户确认
- 系统会优先识别楼栋/房号、面积、业主/租户、电话、店铺、业态、合同日期等基础资料

## 重要说明
- **账单金额可修改**：生成后如果金额不对，点击账单后面的「修改金额」按钮手动调整
- **业主和商户区别**：添加房间时选择「居民」或「商户」，系统自动匹配对应的物业费单价
- **电梯费**：根据房间填写的楼层自动计算，B座7~26层
- **水费特行**：适用于洗车、洗浴、足浴等行业，20.11元/m³

## v1.0 内部使用增强功能

### 公摊分摊

进入「公摊分摊」，录入账期、公摊总金额、收费项目和分摊方式：

- 按面积分摊：按房间面积比例分摊总金额。
- 按户数平均：每个符合条件的房间平均承担。

建议先点击「预览分摊」，确认分摊合计与录入总金额一致后再生成账单。

### 批量更新

进入「批量更新」：

- 可按楼栋/类别批量修改房间物业费单价。
- 可批量修改收费项目默认单价。

执行前系统会显示影响范围并自动备份。

### 合同期控制

生成账单时，系统会跳过合同期外的房间：

- 合同开始日期晚于账期结束：跳过。
- 合同结束日期早于账期开始：跳过。

首页会显示 30 天内即将到期合同和已过期合同数量。

### 对账与欠费分析

「对账报表」中新增：

- 收款对账汇总：按日期、支付方式、操作员、费用类型统计。
- 按费用类型欠费。
- 长期欠费名单。
- 收款 CSV 导出。

### 操作日志

管理员可进入「操作日志」查看关键操作，便于核对金额修改、删除、导入、备份恢复等记录。


### 业态/商户类别

房间资料中已单独增加「业态/商户类别」字段，例如餐饮、零售、美容、办公等。这个字段用于后期按商户类别做表格统计，不再建议只写在备注里。

- 手动新增/编辑房间时，在「业态/商户类别」中填写。
- Excel/CSV 导入中如有「业态」列，系统会单独写入该字段。
- 「备注」仅用于其他补充说明，例如特殊情况、沟通记录等。

## 业主端 H5（本地测试 / 后续小程序前置）

系统已提供业主端 H5 最小闭环，用于后续小程序和在线支付前的业务验证。当前仍运行在桌面 App 本地服务内，默认地址为：

```text
http://127.0.0.1:5001/owner-portal/login
```

如果桌面启动器使用了动态端口，请以桌面窗口打开的本地地址为准，把路径改为：

```text
/owner-portal/login
```

### 当前已支持页面

| 页面 | 用途 |
|---|---|
| `/owner-portal/login` | 业主手机号 + 验证码登录 |
| `/owner-portal/dashboard` | 业主首页，显示房间数、待缴账单、待缴金额 |
| `/owner-portal/rooms` | 查看本人名下房间 |
| `/owner-portal/bills` | 查看本人账单 |
| `/owner-portal/bills/{id}` | 查看账单详情和支付前确认 |
| `/owner-portal/payments` | 查看本人缴费记录 |
| `/owner-portal/logout` | 退出业主端 |

### 本地测试登录流程

1. 打开 `/owner-portal/login`。
2. 输入业主手机号。
3. 点击「发送验证码」。
4. 页面会显示本地测试验证码。
5. 输入验证码后登录。
6. 登录成功后进入 `/owner-portal/dashboard`。

说明：当前验证码用于本地测试，会在页面显示。后续接入真实短信平台后，不再在页面显示验证码。

### 支付前确认

在账单详情页可以输入确认金额并点击「确认金额」。系统会显示：

- 当前欠费。
- 本次确认金额。
- 支付后剩余欠费。
- 是否结清。

当前阶段不会真实入账，也没有接微信支付或支付宝。该功能只用于验证「查账 → 看详情 → 支付前确认」流程。

### 安全边界

- 业主端使用独立 `owner_portal_token`，不复用后台操作员登录状态。
- 业主只能查看本人名下房间、账单和缴费记录。
- 业主端支付前确认会校验账单归属，不能预览其他业主账单。
- 当前本地桌面服务不应暴露到公网。
