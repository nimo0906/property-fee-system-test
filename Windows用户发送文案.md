# Windows 用户发送文案

## 微信短文案

您好，这是「物业收费管理系统」Windows 试用包。

打开方式很简单：

1. 解压我发您的 zip 文件。
2. 打开解压后的文件夹。
3. 双击：

```text
PropertyBillingSystem\PropertyBillingSystem.exe
```

4. 浏览器自动打开后登录：

```text
用户名：admin
密码：admin123
```

如果 Windows 提示安全拦截，请点击：

```text
更多信息 → 仍要运行
```

这是因为当前是内部试用包，还没有做正式软件签名。

如果打开后看到旧测试数据，请先关闭程序，再双击文件夹里的：

```text
清空本机试用数据.bat
```

它会先把旧数据备份到桌面，再清空本机试用数据。

试用时建议重点看：账单管理、物业收费、电子票据、对账报表、业主端页面。试用后麻烦反馈哪些地方看不懂、流程不顺、字段不符合实际。

## 邮件文案

主题：物业收费管理系统 Windows 试用包

您好：

附件是「物业收费管理系统」Windows 试用包，请按以下步骤打开：

1. 解压附件中的 zip 文件。
2. 打开解压后的文件夹。
3. 双击 `PropertyBillingSystem\PropertyBillingSystem.exe`。
4. 浏览器自动打开后，使用以下账号登录：

```text
用户名：admin
密码：admin123
```

如 Windows 安全中心提示拦截，请点击「更多信息」→「仍要运行」。这是因为当前版本为内部试用包，尚未做正式软件签名。

如果打开后看到旧测试数据，说明当前电脑之前运行过本系统。请先关闭程序，再双击 `清空本机试用数据.bat`。该脚本会先把旧数据备份到桌面，再清空本机试用数据。

建议试用时重点查看以下功能：

- 后台首页
- 账单管理
- 物业收费
- 电子票据
- 对账报表
- 业主端页面

试用后请重点反馈：

- 是否能顺利打开。
- 菜单和按钮是否容易理解。
- 收费、账单、对账流程是否符合实际工作。
- 哪些字段、名称或流程需要调整。
- 哪些功能需要在正式使用前补齐。

谢谢。

## 发包前提醒自己检查

发送给用户前，请确认发送的是：

```text
release\windows\物业收费管理系统-v2.0-windows.zip
```

不要单独发送源码目录、`property.db`、`.env` 或 `backups`。
