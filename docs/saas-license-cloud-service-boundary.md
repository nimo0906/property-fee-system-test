# 授权云服务后台边界设计

本文定义 SaaS 云端商业版的授权云服务后台边界。授权服务用于管理客户授权、产品授权、授权校验和授权审计；它是独立服务、独立数据库，不直接承载物业收费业务数据。

## 边界原则

- 授权云服务必须作为独立服务部署。
- 授权云服务必须使用独立数据库，默认库名建议为 `property_license_cloud`。
- 不得与 SaaS 业务主库混用。
- 不得保存客户上传业务数据。
- 不得保存收费对象、业主、账单、收款、导入明细等业务表。
- SaaS 员工后台只调用授权校验接口，只返回授权结果。
- 授权校验结果不得包含业务库连接信息或客户上传文件路径。

## 第一版数据表

| 表名 | 用途 |
| --- | --- |
| `license_customers` | 记录购买系统的客户公司，不记录物业收费业务数据。 |
| `license_products` | 记录可售卖产品，例如员工后台、业主端 H5、支付模块。 |
| `license_entitlements` | 记录客户对产品的授权状态、席位数、到期时间。 |
| `license_audit_logs` | 记录授权创建、变更、校验等操作审计。 |

## 与 SaaS 业务后台的关系

SaaS 业务后台维护租户、项目、员工、收费对象、收费项目、账单、收款、导入和报表。授权云服务只维护商业授权，不写入业务表，不读取客户上传文件。

授权调用返回字段限定为：

- `allowed`
- `status`
- `customer_code`
- `product_code`
- `seats`
- `expires_at`

## 验收口径

- 授权服务 schema 不包含 SaaS 业务表。
- 授权服务接口不返回业务库连接信息。
- 授权服务接口不返回客户上传业务数据。
- SaaS release gate 必须包含授权云服务边界检查。
- 授权审计只记录授权操作摘要，不记录收费明细或客户上传文件内容。
