# SaaS 云端商业版交付演示手册

本文用于正式商业后台交付演示。目标是证明一个新租户从空库开始，可以完成员工后台核心业务闭环，并且租户数据隔离、客户上传数据与系统自身数据隔离。

## 1. 演示范围

- 正式商业后台：登录、租户、项目、权限、收费对象、收费项目、账单、审核、收款、报表、导出、备份和恢复演练。
- 不演示业主端 H5、微信/支付宝真实支付、电子票据、独立授权云服务后台。
- 不使用真实客户数据，只使用脱敏样例数据。

## 2. 新租户从空库开始的验收路径

1. 登录并创建租户会话。
2. 创建项目，确认后续数据都落在当前租户和当前项目下。
3. 导入/录入收费对象，包括住宅房号和商铺铺位号。
4. 配置收费项目，包括面积计费和固定金额计费。
5. 生成账单，校验账期、服务期、应收金额。
6. 账单审核，审核后进入可收款状态。
7. 登记收款，包括全额收款和部分收款。
8. 查看欠费/实收报表，核对应收、实收、欠费。
9. 导出账单或收款记录，确认导出不含内部字段。
10. 备份和恢复演练，确认操作写入审计。

## 3. 隔离要求

- 租户数据隔离：A 公司不能读取 B 公司收费对象、账单、收款、报表、审计和备份记录。
- 客户上传数据与系统自身数据隔离：客户文件保存到客户目录，系统文件保存到系统目录，备份保存到备份目录。
- 页面、导出、日志、审计不得展示生产密钥、数据库密码或内部租户字段。

## 4. 一键演示命令

```bash
python3 scripts/saas_commercial_delivery_drill.py
python3 scripts/saas_commercial_delivery_drill_check.py
```

演示脚本必须输出每个业务步骤的 PASS，并以 `saas_commercial_delivery_drill: PASS` 结束。

## 5. 商业验收门禁

正式交付前必须执行：

```bash
python3 scripts/saas_release_gate.py
```

该门禁会纳入 `scripts/saas_commercial_delivery_drill.py` 和 `scripts/saas_commercial_delivery_drill_check.py`，用于避免演示文档、脚本或页面遗漏。
