UI / COMPONENT REVIEW

物业收费系统页面与组件改造对比小样

结合最新 codex.md，我建议先把“桌面端高频页”的视觉层级、按钮结构、表格密度、表单分组和详情页信息组织统一起来。 目标不是做更花哨的界面，而是让收费员、财务和管理层在高频操作里更快找到重点。

桌面 App 优先 先验证再完成 组件复用 减少重复操作 高频页优先

我建议优先改这 4 类

  • 侧边栏：收拢分组、弱化干扰、突出当前模块。
  • 工具栏：把主操作、次操作、筛选分区，不要一排全堆。
  • 表格 / 详情：提升行内层级，状态、金额、操作要更清晰。
  • 表单：按业务步骤分区，减少“长一坨”式编辑页。

当前更像“功能清单”，建议改成“工作分区 + 当前任务入口”。

重点：减少视觉噪音，突出高频入口。

2. 工具栏与表格区

把“主按钮 / 次按钮 / 筛选条件”分层，不要所有按钮平铺。

重点：减少误触，提升扫描速度。
