# SaaS 云端商业版第一阶段收口报告

## 1. 收口结论

本报告用于 SaaS 云端商业版第一阶段收口。当前阶段定位为员工后台云端化，不包含业主端、真实支付、电子票据和授权云服务后台。第一阶段目标是让不同公司作为独立租户使用同一套云端后台，并确保租户业务数据、客户上传数据和系统自身运维数据保持清晰隔离。

结论：第一阶段已形成可验证闭环，具备进入下一阶段部署演练和客户样例验收的基础。

## 2. 范围对照

| 模块 | 收口状态 | 验收重点 |
| --- | --- | --- |
| 登录、租户、角色权限 | 已纳入 | 平台管理员、租户管理员、财务、收费员、管理层权限边界可测 |

## 3. 数据隔离边界

第一阶段以租户隔离为硬边界。每家公司对应独立 tenant，项目级数据继续带 project_id。业务表、导入批次、账单、收款、审计、备份恢复记录均不得跨租户读取或操作。

客户上传数据与系统自身数据隔离：客户导入文件、预览缓存、错误行文件和确认结果进入租户文件域；系统配置、运行日志、备份元数据、部署资产和运维脚本进入系统文件域。二者不能混放，避免客户资料污染系统自身配置，也避免系统文件误进入客户数据导出。

## 4. 验收闭环

从空租户开始，第一阶段验收路径如下：

```text
创建项目
→ 导入/录入收费对象
→ 配置收费项目
→ 生成账单
→ 账单审核
→ 登记收款
→ 查看欠费/实收报表
→ 导出账单或收款记录
→ 检查审计日志
→ 创建备份记录并提交恢复演练
```

对应自动化命令：

```bash
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 scripts/saas_preflight_check.py
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 scripts/saas_acceptance_check.py
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 scripts/saas_ops_check.py
PYTHONPYCACHEPREFIX=/private/tmp/property_pycache python3 scripts/saas_phase1_closure_check.py
```

## 5. 后置范围

以下内容明确不混入第一阶段，避免破坏当前员工后台闭环：

- 业主端 H5 后置：后续独立规划，不影响当前员工后台验收。
- 微信/支付宝真实支付后置：后续需要支付订单、签名回调、幂等、退款、对账和 HTTPS 公网域名。
- 电子票据平台对接后置：后续单独做发票/票据平台接口边界。
- 授权云服务后台后置：正式授权服务不与当前 SaaS 业务表混在一起。

## 6. 下一阶段建议

下一阶段建议先做部署演练和样例客户验收：使用脱敏样例库，在通用 Linux/VPS 上完成从部署、初始化、导入、出账、收款、报表、备份、恢复演练到日志检查的完整走查。确认稳定后，再进入真实客户试点和更细的业务差异配置。

## 7. 样例客户验收演练

第一阶段收口后新增样例客户验收演练，用于在不接触客户真实资料的前提下验证商业后台闭环。执行 `scripts/saas_demo_tenant_drill.py` 可创建样例物业公司、导入收费对象、配置收费项目、完成账单生成、账单审核、收款登记、对账报表、导出、备份恢复演练和租户隔离复核。详细说明见 `docs/saas-demo-tenant-drill.md`。
