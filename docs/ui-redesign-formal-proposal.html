UI / COMPONENT PROPOSAL

物业收费系统 UI 与组件正式提案

本提案基于你当前的真实页面状态整理，目标不是重做一套新系统，而是保留现有桌面端风格， 在 信息层级、分区方式、表格密度、按钮语义、详情页组织 上做收敛。

桌面 App 优先 高频页优先 物业 / 商业分区 组件统一 先验证再落地
提案结论：
现有视觉基调可以保留，不建议大改品牌色。 重点应放在 更清晰的业务分区 和 更稳定的组件语义。

一、总判断

当前系统已具备统一风格，但高频页面的信息密度仍偏“平均用力”。

现有页面的优点

  • 桌面端层级完整，侧栏、内容区、卡片系统已成型。
  • 整体浅蓝玻璃感统一，视觉辨识度较高。
  • 收费工作台、账单管理、合同档案都有基本的模块边界。

当前最需要改的点

  • 账单页中物业/商业混看，靠目测区分，扫描成本高。
  • 合同档案表格首列信息量大，但分类入口不够硬。
  • 收费工作台的快捷动作与核心经营信息权重接近，主次不够清楚。

四、组件落地建议

下面这些组件最适合接下来统一复用。

推荐优先统一的组件

  • Layout / Sider / Menu：侧栏与导航结构保持统一。
  • Space.Compact：筛选条、日期选择、按钮组合。
  • Table：账单、合同、缴费记录统一表格语义。
  • Tabs / Collapse：物业 / 商业分区与档案分类。
  • Card / Tag / Descriptions：摘要、状态、单据信息块。

落地顺序

  • 优先改 账单管理。
  • 再改 合同档案。
  • 最后微调 收费工作台。
  • 所有页面统一按钮、标签、表头、间距、状态展示。
