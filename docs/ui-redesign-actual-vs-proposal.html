我最终建议的组件方向

和你最新 codex.md 对齐：组件优先参考官方文档，UI 先看成熟产品模式，再落到你系统的业务密度。

优先用的组件

  • Layout / Sider / Menu：保留你现在的窄侧栏基因，但优化分组和当前态。
  • Space.Compact：把筛选区、日期区、按钮区整齐收口。
  • Table：账单、合同、缴费记录统一表格语义，少靠手工拼接。
  • Collapse / Tabs：合同档案和物业/商业分区最适合。
  • Card / Tag / Descriptions：摘要、状态、金额、信息块更适合分层展示。

我建议的改造顺序

  • 先改 账单管理：房间号、归属分区、按钮层级。
  • 再改 合同档案：Tabs + Collapse + 更轻的表头。
  • 最后收紧 收费工作台：保留风格，但强化今日任务。
  • 所有高频页统一：按钮、标签、表格、间距、卡片标题样式。
