# SaaS 云端商业版上线后首日巡检清单

本文用于生产上线后当天和首日巡检，实施人员、运维人员、客户管理员共同确认。目标是尽早发现服务异常、权限异常、租户越权、金额异常、备份异常、日志/审计缺失等问题。

## 巡检时间点

| 时间点 | 重点 | 结论 |
| --- | --- | --- |
| T+0 小时 | 上线完成后立即检查服务、登录、HTTPS、门禁输出 | 通过 / 不通过 |
| T+2 小时 | 抽查业务闭环、租户隔离、日志、审计、备份 | 通过 / 不通过 |
| T+24 小时 | 核对首日业务、报表、备份、客户反馈、回退观察点 | 通过 / 不通过 |

## 服务健康

- 检查 Docker Compose 服务状态。
- 检查 app、PostgreSQL、Nginx、systemd、logrotate。
- 检查 HTTPS 是否可访问。
- 复跑 `scripts/saas_release_gate.py`，确认无 FAIL。

## 登录和权限

- 平台管理员可登录并能查看租户列表。
- 租户管理员只能管理本租户员工账号。
- 财务可出账、审核、收款、查看报表。
- 收费员可收款但不能修改收费项目。
- 管理层只读，不能写入业务数据。

## 租户隔离

- A 公司不能读取 B 公司收费对象。
- A 公司不能读取 B 公司账单、收款、报表、审计、备份记录。
- 核对 `release/saas-isolation-evidence.md`。

## 出账/收款抽查

- 抽查 1 个住宅收费对象和 1 个商业收费对象。
- 核对账期、服务期、应收金额、已收金额、欠费金额。
- 抽查部分收款后欠费是否正确。
- 抽查导出文件不包含内部字段。

## 备份检查

- 执行 `scripts/saas_backup.sh`。
- 抽查备份目录、metadata、checksum。
- 必要时执行 `scripts/saas_restore.sh --verify-metadata` 做恢复演练。

## 日志检查

- 检查 `/var/log/property-saas` 是否正常写入。
- 检查 logrotate 配置是否生效。
- 日志不得出现真实密钥值。

## 审计检查

- 检查登录、出账、审核、收款、导入、备份、恢复演练、高风险操作审计。
- 审计详情必须脱敏。
- 审计必须按租户隔离。

## 报表核对

- 核对应收、实收、欠费。
- 核对账单列表、收款列表、导出文件与报表汇总一致。
- 核对 `release/saas-release-evidence.md` 留档。

## 客户上传数据与系统自身数据隔离

- 客户上传数据只进入客户目录。
- 系统自身数据只进入系统目录。
- 备份和日志分目录保存。
- 不得把客户文件混入应用代码目录。

## 回退观察点

出现以下情况不得上线或必须回退：

- 服务健康检查失败。
- 登录或权限边界异常。
- 租户数据隔离异常。
- 客户上传数据与系统自身数据隔离异常。
- 出账、收款、欠费、报表金额异常。
- 备份检查或恢复演练失败。
- 日志或审计缺失。
- 客户反馈影响核心收费业务。
