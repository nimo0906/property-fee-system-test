1. 参考来源：这次更偏这几类模式

除了组件文档，这次把“成熟后台页面结构”也收进来，避免画得太像展示页。

组件语义

  • Card：单一主题容器。
  • ProCard：卡片、栅格、Tabs 组合。
  • Segmented：分区切换。
  • ProTable：查询 + 表格一体。

数值与状态

  • Statistic：核心数字高亮。
  • Tag：状态与归属标记。
  • Drawer：详情与操作侧边化。

页面模式

  • Dashboard：指标先行。
  • Table：高密度列表。
  • Search bar：筛选前置。

这版结论

  • 主视觉更白、更像后台。
  • 减少大块渐变、装饰性标题。
  • 高频页统一为：筛选条 + 统计卡 + 表格 + 抽屉。

7. 建议实施顺序

确认 v3 后，再按高频程度逐页落地，保持改一页、测一页、看一页。

第一批：高频操作

物业收费、商业收费、数据导入、业主管理、房间管理。

第二批：财务核对

缴费记录、发票管理、对账报表、期末结账。

第三批：规则与自动化

收费项目、抄表管理、批量更新、自动出账、公摊分摊。

第四批：系统维护

操作日志、备份、用户、健康、更新、试用数据清空。
