# 2.0 桌面稳定版收口冻结记录

## 冻结结论

- 冻结版本：v2.0 桌面稳定版收口
- 冻结日期：2026-06-13
- 当前主线：停止新增业务板块，进入现有模块统一优化、真实使用防错、交付冻结。
- 云端口径：云端数据底座保留为技术备查，仅供管理员/技术人员查看，不作为当前开发主线。
- 交付状态：以本地桌面 App 正式可用为目标，优先保障启动、数据、金额、权限、导入、报表、打包检查。

## 本轮收口范围

- 现有模块统一优化：业主管理、房间管理、收费项目、抄表、公摊、物业收费、商业收费、空间合同档案、账单、缴费、发票、报表、导入、期末结账。
- 真实使用防错：按员工日常顺序核对“资料录入/导入 → 抄表 → 出账 → 账单核对 → 收费 → 收据/发票 → 报表 → 期末结账”。
- 金额口径复核：面积 × 单价、周期月数、水电抄表、公摊分摊、自动出账、商业收费、物业收费和合同档案展示保持一致。
- 隐藏/废弃入口清理：不属于当前桌面稳定版范围的模块不作为员工日常入口。
- 隐藏/保留模块清单：详见 `docs/hidden-module-policy.md`，维修、停车、押金保持菜单隐藏且直连 404；业主端和支付订单保持下线 404。
- 云端数据底座保留为技术备查：保留 schema、seed SQL、迁移摘要等页面，但不引导员工按云端上线推进。

- Python 编译：通过（2026-06-13）。
- 全量测试：441 passed, 4 skipped（2026-06-13）。
- 桌面发布资源检查：通过（2026-06-13）。
- `scripts/desktop_release_check.py`：通过。

- 数据一致性只读扫描：通过，未发现负金额账单、断链账单、合同日期倒挂、缺失房号等硬错误。
- 金额专项修复：自动出账不再将 `物业费(商业)` 套用到 B座商户房间；`合同租金/合同物业费/合同押金` 不再显示在物业/商业日常收费入口。
- 导入专项验收：通过，统一导入工作台、商业合同导入、B座出租合同导入、预览核对、确认前备份和旧合同导入口跳转均有回归覆盖；真实商业合同 Excel 只读解析通过，百分比/销售额租金会进入问题行人工核对，不再静默当作面积单价导入。
- macOS 桌面交付继续按 `.app` 验证；Windows 正式包继续在 Windows 电脑上运行打包脚本并验收。

## 冻结后的变更规则

冻结后只接受 P0/P1 修复：

1. P0：无法启动、数据丢失、金额错误、账期错误、越权、误删、桌面包不可用。
2. P1：主流程阻塞、筛选/分页明显错误、状态错误、导出/打印明显错误、权限菜单与实际权限不一致。

以下内容不进入本次稳定版主线：

- 新增业务板块。
- SaaS / 云端上线 / 小程序 / 业主端支付等新范围。
- 发票真实税务平台对接。
- 更深度经营驾驶舱扩展。
- 自动更新发布流程增强。
- Windows 正式签名和安装包形态。
