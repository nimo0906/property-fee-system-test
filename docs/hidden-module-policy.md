# 2.0 稳定版隐藏/保留模块清单

## 当前主线

2.0 稳定版只保留桌面收费主流程：资料录入/导入、抄表、物业收费、商业收费、自动出账、账单核对、缴费、收据/发票、报表、期末结账、合同档案、备份和系统维护。

## 保留但禁用

以下模块不进入当前员工日常入口，菜单隐藏，直连必须返回 404：

- 维修管理：`/repairs`
- 停车管理：`/parking`
- 押金管理：`/deposits`

处理原则：暂不删除数据库表和历史代码，避免破坏旧库、健康检查、清空试用数据和未来兼容；如未来重新启用，必须重新做权限、菜单、金额和回归测试。

## 已下线

以下模块不属于当前桌面稳定版，保持 404：

- 业主端：`/owner-portal`
- 支付订单：`/payment_orders`

处理原则：不在后台菜单展示，不作为隐藏入口保留；未来如果做手机端或真实支付，应重新设计独立模块。

## 技术备查

以下入口保留给管理员/管理层，不作为员工日常操作主线：

- 云端技术备查：`/cloud_schema`
- 交付中心：`/delivery_center`
- 系统健康：`/system_health`
- 系统更新：`/system_update`
- 备份恢复：`/backups`
- 清空试用数据：`/trial_data_reset`

处理原则：菜单和直连权限必须同时受控；后续稳定版冻结后只接受 P0/P1 修复。
