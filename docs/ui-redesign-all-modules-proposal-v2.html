菜单其他模块：更像成熟后台产品的 UI 升级小样

这版把样式收得更稳：先从 Ant Design / ProComponents 的组件语义出发，再叠加 Pageflows / 真实后台的常见信息结构。目标不是“更花”，而是更像可直接落地的桌面财务系统。

Ant Design / ProComponentsTable + DrawerStatistic + CardPageflows 模式参考少装饰，多执行

1. 这版更重点参考的组件与页面模式

这次加大了对“卡片、分段控制、抽屉、紧凑间距、统计卡”的权重。

Ant Design / ProComponents

  • Card：单主题内容容器。
  • ProCard：Card / Tabs / Grid 融合。
  • Segmented：切换关联区域内容。
  • ProTable：查询 + 表格一体。

状态与数据呈现

  • Statistic：高亮关键数字。
  • Drawer：不离开当前页做副任务。
  • Space.Compact：紧凑连接筛选控件。

后台页面模式

  • Dashboard：先指标、后任务。
  • Table：密集列表与操作密度。
  • Search bar：筛选条更靠前。

这版收敛出的结论

  • 页面标题更轻，先让数据和动作说话。
  • 表格尽量一页一职责，详情进 Drawer。
  • 切换用 Segmented / Tabs，不再靠散按钮。

2. 全模块组件地图（v2 收敛版）

这里不再追求每页都“有戏”，而是统一语义和密度。

物业收费 / 商业收费

TableDrawerStatistic

分段切换 + 左表右抽屉，减少在表格里找动作。

数据导入

UploadSteps

上传、预览、问题行、确认导入，四步清楚分离。

业主管理 / 房间管理

ProTableDrawer

主表只管索引，详情、合同、历史放抽屉。

收费项目 / 抄表管理

Editable TableTag

高频规则项统一放表格里编辑，状态用 Tag。

自动出账 / 公摊分摊

StepsConsole

规则、预览、执行、历史，做成控制台式流程。

缴费记录 / 发票管理

LedgerTabs

按状态分区，金额、发票状态、回溯动作固定。

对账报表 / 期末结账

StatisticChecklist

先看结果，再看差异，再做结账确认。

系统维护

SettingsAudit

健康、备份、日志、用户、更新做成一个维护面板。

7. 我建议的执行顺序

确认 v2 样式后，按使用频率从高到低逐页落地。

第一批：高频操作

物业收费、商业收费、数据导入、业主管理、房间管理。

第二批：财务核对

缴费记录、发票管理、对账报表、期末结账。

第三批：规则与自动化

收费项目、抄表管理、批量更新、自动出账、公摊分摊。

第四批：系统维护

操作日志、数据备份、用户、健康、更新、试用数据清空。

确认后我就先改第一批，并保持“改一页、测一页、看一页”的节奏。
