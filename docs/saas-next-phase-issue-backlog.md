# SaaS 云端商业版下一阶段可执行工单清单

本文把 `docs/saas-next-phase-decision-checklist.md` 中的候选方向拆成可独立排期、开发、验收的工单包。只有在 `docs/saas-day7-stability-review.md` 通过后，才建议选择其中一个方向进入实施。

## 工单拆分原则

- 每个工单必须有明确验收口径、依赖关系、风险边界。
- 不得与 SaaS 业务主库混用。
- 不得破坏租户数据隔离。
- 不得把客户上传数据与系统自身数据混在一起。
- 不把业主端、支付、授权、桌面业务迁移混在同一个提交或同一个验收口径里。

## 跨工单验收口径

- 每个工单必须有 focused tests。
- 每个工单完成后必须跑 SaaS release gate。
- 涉及桌面共用逻辑时必须跑桌面回归门禁。
- 涉及数据结构时必须先明确租户和项目隔离口径。
- 涉及文件上传、对账、授权文件时必须复用客户上传数据与系统自身数据隔离原则。

## 优先级建议

1. 首选：真实客户强需求且能付费的方向。
2. 次选：能复用当前员工后台收费闭环且风险低的方向。
3. 后置：需要外部平台资质、回调公网、复杂资金流或独立授权体系的方向。
